- **Read the methodology**: see [methodology.md](methodology.md) for how ranks are computed and what each metric means.
- **Read the Elite program design**: see [elite-program.md](elite-program.md) for why the leaderboard is split into public practice and hidden ranked, and how hardening works.
- **Submit a method**: the submission protocol is documented in [submitting.md](submitting.md). Submissions open once the evaluation runner goes live.
- **Build a method locally**: every tool you need to develop a compactor is already in the Python package — see [writing-a-compactor.md](writing-a-compactor.md).
